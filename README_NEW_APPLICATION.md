# Модуль «Нова заявка»

Додано до кабінету складу розділ комплектування магазину за штрихкодами.

## Запуск

```bash
npm start
```

Render використовує `process.env.PORT`. Дані зберігаються у `DATA_DIR/db.json`, якщо `DATA_DIR` задано, інакше у локальній папці `data`.

## Зміни даних

Під час читання існуючого `db.json` безпечно ініціалізуються нові масиви:

- `productBarcodes`
- `applications`
- `applicationLogs`

Існуючі масиви та дані не очищаються.

## Основні маршрути

- `/admin-applications` — список і створення заявок
- `/admin-applications?id=...` — комплектування
- `/admin-applications/:id/print` — друкована накладна
- `/admin-applications/:id/pdf` — відкриття друкованої накладної з діалогом збереження у PDF
- `/admin-applications/:id/xlsx` — Excel

## Камера

Автоматичне розпізнавання використовує браузерний `BarcodeDetector`. Там, де браузер його не підтримує, залишається ручне введення та USB/Bluetooth-сканер, який працює як клавіатура.

## Оновлення сканера камери

Сканування камерою перероблено для мобільних браузерів iPhone та Android через ZXing Browser із двома CDN-джерелами. Підтримано запуск задньої камери, `playsinline` для Safari, зупинку потоку, захист від повторного зчитування одного штрихкоду та зрозумілі повідомлення про відмову в доступі.

Для роботи камери сайт потрібно відкривати через HTTPS (Render надає HTTPS). Під час першого запуску користувач має дозволити браузеру доступ до камери.
